# Isomer Generator for Shanghai Gaokao

Constraint-based organic isomer generator for Shanghai Gaokao, refactored for carbonyl-aware constraint handling and lower-bound preparation logic.

The program accepts a chemical formula and high-school chemistry style constraints, generates all mathematically valid molecular structures, filters chemically unreasonable ones, and outputs the legal isomers and their structures.

The project focuses on interpretable rule-based enumeration, not machine learning.

---

# Features

The program can:

- Enumerate possible constitutional isomers
- Apply high-school chemistry constraints including:
  - functional group counts
  - carbonyl-aware constraint handling (shared carbonyl center)
  - chiral centers
  - number of unique hydrogens
  - hydrogen NMR peak ratios
  - benzene substitution patterns
- Filter chemically unstable structures
- Render molecular structures from SMILES
- Provide a web interface (Streamlit) for interactive use

Output includes:

- number of valid isomers
- structure drawings of all valid isomers

---

# Scope and Limitations

This project intentionally targets the scope of Shanghai Gaokao chemistry problems.

Current limitations:

1. Only molecules containing one aromatic core (benzene) are supported.
2. Only atoms C, H, O, N, Cl, Br are allowed.
3. No rings other than the aromatic core are generated.
4. Only chemically stable structures commonly discussed in high school chemistry are retained.
5. Visualization is provided through the web interface.
6. Enumeration may become slow for large molecular formulas due to combinatorial explosion.

7. Carbonyl-containing functional groups are internally unified as a shared carbonyl center during generation and resolved during filtering.

---

# Input Parameters

The generator accepts the following constraints:

1. Chemical formula
2. Functional group counts  
   (can be specified as null, not null, or an integer)
3. Number of chiral centers
4. Number of unique hydrogens
5. Ratio of hydrogens in different environments (NMR peak area ratio)
6. Substitution pattern on the benzene ring
7. Number of unique hydrogens on the benzene ring

# Pipeline Overview

## 1. preparation.py

This module prepares the generation plan using a lower-bound strategy.

Main tasks:

1. Compute omega (ω) from the chemical formula  
   ω represents the structural degree of unsaturation.

2. Convert user constraints into generation-relevant lower bounds  
   - Carbonyl-related constraints (ester, aldehyde, amide, etc.) are merged into a shared "carbonyl" requirement  
   - "Existence" constraints (e.g. 有) are treated as ≥1

3. Enumerate possible structural strategies under these lower bounds

This stage prioritizes pruning the search space early rather than enforcing exact functional group counts.

---

## 2. molecule_generator.py

This module performs structure generation using a unified extension model.

Key idea:

- All carbonyl-containing functional groups are represented by a single "C=O" block
- The identity of ester / aldehyde / ketone / amide is NOT fixed during generation

Molecule Construction:

- Structures are generated by attaching extension blocks
- The carbonyl carbon remains an active extension site
- Final functional group identity is determined later in filtering

This design avoids premature constraint conflicts and enables shared carbonyl structures.

---

## 3. filter.py

Note:

Functional group identities (ester, aldehyde, amide, etc.) are determined here using SMARTS matching, rather than during generation.

A single carbonyl center may satisfy multiple high-school functional group definitions (e.g. formate esters counted as both ester and aldehyde).

This module performs chemical filtering.

Steps:

1. Remove unstable or chemically unreasonable molecules
2. Calculate molecular features required by user constraints
3. Apply constraint checks to keep only legal isomers

Examples of computed properties include:

- functional group counts
- number of chiral centers
- unique hydrogen environments
- hydrogen NMR ratios
- benzene substitution patterns

---

## 4. renderer.py

Responsible for visualization.

Functions:

- convert SMILES strings to RDKit molecules
- render molecular structures as images

---

## 5. ui.py

Implements the Streamlit web interface.

Functions:

- collect user input
- convert user constraints into preparation constraints (lower-bound based)
- run the generation pipeline
- display resulting structures (with a fixed display cap for performance)
